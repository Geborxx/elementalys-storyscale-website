StoryScale docs

Controlled publishing to Jira.

StoryScale writes Story Points only when estimates are complete and the user confirms publication.

Overview Getting Started Estimation Method Permissions & Data Publishing FAQ Support Release Notes

Publish Requirements

  • The ranking must be complete.
  • Guided calibration, when active, must be finished.
  • A Story Points field must be resolved.
  • The user must confirm the publish action.

What Gets Updated

Only current session issue estimates are sent to Jira. StoryScale updates the configured Story Points field with the final approved values.

Historical anchors are excluded from publishing by design.

Error Handling

If Jira rejects an issue update, StoryScale reports partial success and identifies failed items. Common causes include field configuration, permissions, workflow restrictions, or issue editability.
