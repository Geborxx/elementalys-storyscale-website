Getting started.

Run a StoryScale session from a Jira project page using a sprint, JQL scope, or single issue lookup.

Overview Getting Started Estimation Method Permissions & Data Publishing FAQ Support Release Notes

Before You Start

  • Use Jira Cloud with the StoryScale Forge app installed.
  • Open StoryScale from a Jira project page.
  • Make sure the project has a Story Points number field available.
  • Choose a bounded scope: an active sprint, future sprint, JQL query, or a single issue key.

Start A Session

  1. Open StoryScale from the Jira project navigation.
  2. Select Sprint import or JQL import.
  3. Load issues and review readiness signals.
  4. Start comparing stories by choosing which item is more complex.
  5. Review the final estimation board.
  6. Publish only when the team agrees with the estimates.
Tip: for JQL, keep the scope small enough for a real planning conversation. A sprint-sized set usually works best.

Supported Import Patterns

SprintLoad active or future sprint issues from the current Jira context.
JQLUse a bounded Jira query with autocomplete support and validation.
Single issueUse natural lookup formats like AO-201, AO 201, key=AO-201, or ID=AO 201.
